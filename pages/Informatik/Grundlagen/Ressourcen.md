## Kursbeschreibung

### Grunddaten

- **Gesamtwochenstunden**: 8
- **Dauer**: Ca. 4 Jahre (Lehrberuf)
- **Minimum Testanzahl**: 3 Tests
- **Test-Policy**: Eine Note wird nicht gerechnet

### Progression

**Woche 1-4**: Grundstrukturen (If-Else, Schleifen, Variablen)
**Woche 5-8**: Datenstrukturen (Arrays, Listen, Maps)
**Monat 2-3**: Objektorientierung (Klassen, Vererbung)
**Monat 4+**: Fortgeschrittene Muster und Anwendungen

### Schwierigkeit

```
Leicht:     ▓░░░░ (Week 1-2)
Mittel:     ▓▓▓░░ (Week 3-8, Monat 2-3)
Schwer:     ▓▓▓▓░ (Monat 4+)
Expert:     ▓▓▓▓▓ (Projekte, Optionales)
```

**Realität**: Eine 1er Note nur mit konsistenter Mitarbeit!

## Erfolgreiche Strategien

### 1. Mitarbeit

- **Frag Fragen!** Zu verstehen ist besser als Angst zu haben
- Keine dummen Fragen, nur dumme Antworten (die du nicht fragst)
- Besuch jede Stunde

### 2. Üben

- Code ist nicht Theorie, es ist Praxis
- Schreib Code, mach Fehler, lerne draus
- GitLab/GitHub Push für jede Übung

### 3. Testen

- Bevor Tests anstehen, wiederhole
- Old Exams wenn vorhanden
- Mit Mitschülern üben

### 4. Review & Refactor

- Schau alten Code an
- Kannst du es jetzt besser machen?
- Code Review mit Peers

## Lernziele (High Level)

Nach diesem Kurs kannst du:

- ✓ Algorithmen in Notationssprachen modellieren (PAP, Pseudocode)
- ✓ Kontrollstrukturen richtig nutzen (If, Schleifen)
- ✓ Datenstrukturen auswählen (Array, List, Map, etc.)
- ✓ OOP Konzepte verstehen (Klassen, Vererbung, Polymorphismus)
- ✓ Code schreiben, testen, debuggen
- ✓ Git und Versionskontrolle nutzen
- ✓ Professional Tools (Maven, Docker, IDE) verwenden

## Prüfung

### Format

- Multiple Tests kapitelweise
- Mindestens 3 Tests
- Eine Note wird nicht gerechnet (Kulanz)
- Final: Praktische Umsetzung

### Was wird geprüft

- Konzeptuelles Verständnis (nicht nur Code)
- Algorithmen-Design
- Datenstrukturen-Auswahl
- Code-Implementierung
- Testing und Debugging

## Tipps für Erfolg

1. **Early Bird**: Fragen stellen sobald unklar
2. **Regelmässig üben**: 20min täglich > 3h am Wochenende
3. **Code Review**: Von anderen lernen
4. **Fehler akzeptieren**: Sie sind Lernchancen
5. **Dokumentieren**: Dein Lernen hilft anderen
